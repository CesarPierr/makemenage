# Réécriture des textes — ton plus naturel, simple et adoptable

## 1. Intention

Le texte de l'app doit arrêter de "décrire des fonctionnalités" et commencer à "aider quelqu'un à faire ses tâches".

Le ton cible :

- simple ;
- direct ;
- chaleureux sans être enfantin ;
- utile ;
- jamais bureaucratique ;
- jamais trop technique dans le flux quotidien.

## 2. Principes de rédaction

### 2.1 Préférer l'action

- dire ce que la personne peut faire maintenant ;
- éviter les descriptions abstraites ;
- éviter les phrases qui vendent la structure interne de l'app.

### 2.2 Préférer le concret

- `À faire maintenant` plutôt que `Priorités`;
- `Faire plus tard` plutôt que `Reporter à une autre date` quand le contexte est clair ;
- `Qui s'en occupe` plutôt que `Attribution`.

### 2.3 Garder le vocabulaire technique hors du flux principal

- éviter `récurrence`, préférer `routine` ;
- éviter `occurrence`, préférer `cette fois` ;
- éviter `charge`, préférer `temps prévu` ou `répartition`.

## 11. Onboarding

## 11.1 Objectif

Réduire le texte et éviter le discours "fonctionnalités".

## 11.2 Proposition

### Écran 1

- Titre : `Bienvenue`
- Texte : `On met l'essentiel en place en 2 minutes.`
- CTA : `Commencer`

### Écran 2

- Titre : `Choisis quelques routines`
- Texte : `Prends une base simple. Tu ajusteras le reste ensuite.`
- CTA : `Ajouter ces tâches`
- CTA secondaire : `Passer`

### Écran 3

- Titre : `Invite les autres quand tu veux`
- Texte : `Tu peux continuer seul maintenant et ajouter les autres plus tard.`
- CTA : `Continuer`

### Écran final

- Titre : `C'est prêt`
- Texte : `Tu peux maintenant voir ce qu'il y a à faire aujourd'hui.`
- CTA : `Voir mes tâches`

## 13. Microcopy systèmes

### Succès

- `C'est enregistré.`
- `Tâche terminée.`
- `Changement pris en compte.`
- `Routine mise à jour.`
- `Absence ajoutée.`
- `Le planning a été revu.`

### Erreurs

- `Impossible d'enregistrer pour le moment.`
- `Cette tâche n'a pas pu être mise à jour.`
- `Le changement n'a pas été appliqué.`
- `Réessaie dans un instant.`

### Confirmations

- `Retirer cette routine du planning futur ?`
- `Annuler seulement cette tâche ?`
- `Écraser les changements faits à la main sur les prochaines tâches ?`

## 14. Règle finale

Si un texte ressemble à :

- une doc de fonctionnalité ;
- une justification d'architecture ;
- une phrase marketing vague ;

alors il faut le réécrire.

Le bon texte doit aider l'utilisateur à :

- comprendre tout de suite ;
- agir sans hésiter ;
- ne pas avoir peur des conséquences.
